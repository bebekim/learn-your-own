# Recording Contract

The observer captures evidence for later learning. It does not decide what the
system learned.

Required buckets:

- run identity: workspace, repo family, branch, commit, Bead, and spec
- goal: success criteria, expected process, stop condition, and risk class
- execution context: task shape, functional axis, domain axis, stack, tools,
  files, and commands
- model usage: model, provider, role, reasoning effort, local/remote status,
  token counts, estimated cost, latency, routing reason, and escalation path
- outcome: tests, checks, verification result, review findings, corrections,
  missing ingredients, and guardrail result
- trace: important events and state transitions

Good observation says what happened and where the evidence lives.
Bad observation says what should change next.
